# Contrast Finance Backend v0.37.17 — deferred manager delete and restore

## Что изменено

Уточнена логика удаления менеджера из админского «Обзора».

Теперь удаление менеджера — это не исчезновение из текущего месяца, а временное отключение до конца месяца:

- админ нажимает красный `×` возле менеджера;
- кабинет менеджера блокируется сразу через `users.is_active = false`;
- в текущем месяце менеджер остаётся в «Обзоре» серой строкой;
- вместо красного удаления появляется маленькая зелёная кнопка восстановления `↺`;
- восстановление доступно только до конца месяца удаления;
- со следующего месяца, если менеджер не восстановлен, он больше не отображается в списке менеджеров в «Обзоре»;
- мероприятия, заявки и фактический доход остаются в базе и продолжают участвовать в планах отдела/компании.

## Техническая логика

Новых миграций нет.

Используются существующие поля:

- `users.is_active` — блокирует кабинет менеджера;
- `users.updated_at` — фиксирует месяц отключения менеджера.

## Изменения в файлах

- `app/web/app.js`:
  - админ теперь загружает пользователей с `include_inactive=true`;
  - в «Обзоре» показываются активные менеджеры и отключённые менеджеры только за месяц их отключения;
  - для отключённого менеджера строка получает серый режим;
  - вместо удаления выводится кнопка восстановления `↺`;
  - добавлена отправка `POST /users/{user_id}/restore`.
- `app/web/styles.css`:
  - серый стиль строки отключённого менеджера;
  - зелёная кнопка восстановления;
  - placeholder для случая, когда срок восстановления уже прошёл.
- `app/api/routes/users_manage.py`:
  - `DELETE /users/{user_id}` оставляет soft delete;
  - новый `POST /users/{user_id}/restore` восстанавливает менеджера только в месяце удаления.
- `app/web/index.html`: cache-bust обновлён до `app.js?v=0.37.17` и `styles.css?v=0.37.17`.
- `app/core/config.py`: VERSION = `0.37.17`.

## Backend / DB

Новых миграций нет.
