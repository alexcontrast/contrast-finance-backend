# Contrast Finance Backend v0.38.4

Хотфикс доступа к карточке мероприятия в read-only кабинете главдепа после v0.38.3.

## Что исправлено

- Главдеп мог видеть мероприятие в списке отдела, но при открытии карточки получать `No access to this event`.
- Причина: список главдепа уже учитывал доли отдела и менеджеров отдела, а общий guard открытия карточки проверял только `event.department_id` и связь через `event_shares` из ORM-relationship. Для старых/перекинутых мероприятий это было недостаточно.
- Теперь проверка доступа для `department_head` дополнительно учитывает:
  - основной отдел мероприятия;
  - отдел основного менеджера мероприятия;
  - доли `event_shares`, найденные прямым SQL-запросом через менеджеров отдела.
- Все endpoints карточки мероприятия (`/events/{id}`, `/summary`, `/items`, `/payment-requests`) передают `db` в общую проверку доступа.

## Технически

- Изменены: `app/services/authorization.py`, routes карточки мероприятия, версия и cache-bust.
- БД и миграции не менялись.
- Проверки:
  - `node --check app/web/app.js`
  - `python3 -m compileall -q app`
