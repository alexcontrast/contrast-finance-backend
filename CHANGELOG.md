# Changelog

## v0.5.44 — Block payment requests for archived events

- Для менеджеров запрещено создавать новые заявки на оплату по архивным мероприятиям: `status=accepted` и `money_status=cash_received`.
- Правило проверяется на backend для обычных заявок по позициям и заявок на ЗП менеджера.
- В Telegram-боте архивные мероприятия исключены из списка выбора для новой заявки; дополнительная серверная проверка сохранена на случай старого открытого сценария.
- В кабинете менеджера кнопка `Оплатить` у архивного мероприятия отключена с понятной подсказкой.
- Администраторские операции не ограничены этим правилом.
- Миграций БД нет.
