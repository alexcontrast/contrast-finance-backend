# Contrast Finance v0.5.56

## НДС подрядчика после проверки КГД

- Найдена причина: обработчик единого API КГД определял плательщика НДС только по нескольким словам из локализованного поля `vatInfo`, но игнорировал отдельное официальное поле `vatDate` — дату постановки на учёт по НДС.
- Из-за этого ОУР распознавался правильно и давал 10% налоговых вычетов, однако при другой формулировке или казахском тексте `vatInfo` компания ошибочно получала статус `our_no_vat`, поэтому НДС в смете оставался нулевым.
- `vatDate` теперь подтверждает действующую регистрацию по НДС. Явные статусы снятия, прекращения или аннулирования регистрации имеют приоритет и не дают НДС-вычет.
- Добавлено распознавание русских, казахских и английских вариантов статуса. Неизвестный ответ больше не считается автоматически «без НДС»: для него используется резервная проверка КГД.

Расчётные ставки, интерфейс, база данных и миграции не менялись.
