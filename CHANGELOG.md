# Contrast Finance v0.5.52

## KGD technical diagnostics

- Техническая недоступность КГД больше не маскируется как «БИН не найден».
- Отсутствующий `KGD_API_KEY` возвращает явную ошибку настройки.
- Если основной и резервные endpoint КГД не отвечают, сайт получает HTTP 503 с конкретными HTTP-статусами.
- В Railway логируются безопасные диагностические данные без раскрытия токена.
- Telegram-бот показывает ту же конкретную техническую ошибку.
- При техническом сбое КГД существующие НДС и вычеты больше не обнуляются как будто контрагент не найден.

# CHANGELOG

## v0.5.51

### Исправлено

- В кабинете менеджера нажатие «Добавить позицию» больше не очищает уже заполненную строку сметы.
- Перед любым структурным действием таблицы текущие значения всех видимых строк принудительно синхронизируются из DOM в модель черновика.
- Та же защита добавлена для переключения внешней/внутренней сметы, Enter с добавлением новой строки, drag-and-drop и удаления позиции.
- Сохраняются наименование, цена, количество, дни, факт, способ оплаты, БИН, примечания и остальные редактируемые поля.

### Причина

Некоторые действия полностью перерисовывали таблицу, полагаясь только на события `input/change`. Если последнее значение ещё не успевало попасть в модель черновика или поле теряло фокус одновременно с нажатием кнопки, новая разметка строилась из старых данных и визуально очищала строку.

### Технически

- Изменён только frontend: `app/web/app.js` и cache-bust в `app/web/index.html`.
- Backend, база данных и миграции не менялись.
