# Contrast Finance Backend v0.37.21

Changed files only.

## Что изменено

Финальная сборка вкладки **«Закрыть месяц»**:

- вкладка стала рабочей компактной калькуляцией месяца, а не заглушкой;
- добавлен минималистичный ввод расходов месяца: название, сумма, распределение, комментарий;
- распределение расхода **«По плану месяца»** берёт проценты Санжар/Рауфаль из вкладки **«Задать планы»** именно выбранного месяца;
- для расходов доступны варианты: по плану месяца / 100% Санжар / 100% Рауфаль / вручную;
- добавлен компактный список расходов с удалением;
- добавлен расчёт закрытия по отделам: план, факт, расходы, процент руководителя, ЗП руководителя, остаток;
- проценты руководителей сохранены: 10% / 15% при выполнении плана;
- учредители названы по именам: Александр, Дмитрий, Иван;
- кнопка **«Закрыть месяц»** сохраняет snapshot калькуляции в `monthly_closings`;
- кнопка **«Открыть месяц»** переводит закрытие в статус `reopened`;
- кнопка **«Пересчитать»** обновляет расчёт, а если месяц уже закрыт — пересохраняет закрытый snapshot;
- закрытие месяца не блокирует редактирование мероприятий и заявок;
- восстановление удалённого менеджера дополнительно запрещается, если месяц его удаления уже закрыт;
- закрытие считает доходы отделов с учётом соавторств `event_shares`, как в админском обзоре;
- статус `reopened` в админке больше не считается закрытым месяцем.

## Backend

Изменены существующие endpoints и добавлены новые без миграций:

- `POST /monthly-closings/close?month=YYYY-MM`
- `POST /monthly-closings/reopen?month=YYYY-MM`
- `DELETE /monthly-expenses/{expense_id}`

Новых таблиц и миграций нет.
