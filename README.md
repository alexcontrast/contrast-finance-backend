Contrast Finance Backend v0.35.64 — changed files only

Доработка кнопки `Оплатить`: отделение проверки БИН от создания заявки.

Точная причина:
- в v0.35.63 проверка БИН выполнялась внутри `Создать заявку`
- из-за этого пользователь не видел промежуточный результат:
  способ оплаты/БИН/НДС/вычеты уже записались в смету или нет
- после создания заявки модалка закрывалась, но не было понятного подтверждения
- при повторной заявке поле БИН не блокировалось явно в модалке оплаты

Исправлено:
1. В модалке оплаты для способа `По счету` добавлена кнопка `Проверить` рядом с БИН.
2. `Проверить`:
   - сохраняет позицию
   - запускает `/event-items/{item_id}/tax/check`
   - фиксирует за позицией:
     - способ оплаты `invoice`
     - БИН
     - налоговый статус
     - НДС
     - вычеты
   - сразу обновляет строки сметы/итоги/миникарточку на экране
3. `Создать заявку` по счету теперь не запускает КГД.
   Оно требует уже проверенный и зафиксированный БИН.
4. Если БИН уже зафиксирован за позицией:
   - поле БИН неактивно
   - кнопка `Проверить` неактивна
   - заявку можно создать сразу
5. После успешного создания заявки:
   - модалка закрывается
   - показывается toast `Заявка отправлена`
   - dashboard и карточка мероприятия обновляются

Backend не менялся относительно v0.35.63.
Миграций нет.

Проверки:
- app.js прошёл node --check
- Python-файлы компилируются
