Contrast Finance Backend v0.35.39 — changed files only

Фикс рассинхрона кнопки КГД в новой/редактируемой строке.

Проблема:
- визуально в строке стоял способ оплаты `По счету`
- но draft-состояние строки иногда ещё хранило старый/пустой payment_method
- КГД-кнопка проверяла именно draft item, а не фактическую строку в DOM
- поэтому после проверки кнопка могла остаться галочкой, вычеты не обновлялись,
  а следующий клик писал “проверка доступна только По счету”

Исправлено:
- перед КГД-проверкой строка синхронизируется из DOM в draft:
  payment_method, BIN/ИИН, факт
- для временной tmp-строки КГД сначала сохраняет позицию в базу,
  получает настоящий id, потом запускает КГД
- после успешной КГД-проверки карточка перерисовывается,
  чтобы галочка стала карандашом и row id стал настоящим
- payment_method в строке после обновления явно сохраняется как `invoice`
- при смене способа оплаты строка тоже синхронизируется перед перерисовкой
- app.js проверен через `node --check`

index.html подключает app.js/styles.css с ?v=0.35.39

Миграций нет.
