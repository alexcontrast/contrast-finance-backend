Contrast Finance Backend v0.37.11 — changed files only

Фикс логики `Архив мероприятий`.

Проблема:
- мероприятие со статусом `На проверке` и `Деньги в кассе`
  попадало в архив мероприятий.

Причина:
- frontend считал архивом любое мероприятие с money_status=cash_received,
  если его status не draft/revision/cancelled.
- status=review не был исключён, поэтому `На проверке + Деньги в кассе`
  ошибочно уходило в архив.

Исправлено:
- архив мероприятий теперь определяется только так:
  status = accepted / Принято
  И money_status = cash_received / Деньги в кассе.

Теперь:
- Черновик + Деньги в кассе → Мероприятия
- На доработке + Деньги в кассе → Мероприятия
- На проверке + Деньги в кассе → Мероприятия
- Принято + Деньги в кассе → Архив мероприятий

Миграций нет.
Backend не менялся относительно v0.37.10.

Проверки:
- app.js прошёл node --check
- Python-файлы компилируются
