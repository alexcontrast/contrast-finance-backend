Contrast Finance Backend v0.35.84 — changed files only

Фикс скорости проверки БИН в модалке оплаты и сохранения суммы заявки.

Точная причина долгой проверки:
- в модалке `Проверить БИН` делало:
  1. saveDraftEvent()
  2. saveDraftItems() — сохранение всех строк сметы
  3. /tax/check
  4. снова saveDraftEvent()
  5. снова saveDraftItems() — повторное сохранение всех строк сметы
- в смете проверка была быстрее, потому что там не было второго полного сохранения после КГД

Что исправлено:
- второе полное сохранение после `/tax/check` убрано
- `/tax/check` сам уже записывает в backend:
  - iin_bin
  - iin_bin_locked
  - tax_check_status
  - vat_amount
  - deduction_amount
- frontend только обновляет локальную позицию и экран

Нюанс:
- если в модалке выбрана `+ создать новую позицию`, первое сохранение всё равно нужно,
  потому что сначала надо создать строку позиции в БД, и только потом к ней можно привязать КГД-проверку

Также исправлено:
- сумма заявки больше не стирается после успешной проверки БИН
- раньше она стиралась из-за renderPaymentPositionState(), где amountInput.value сбрасывался в пустое значение

Backend не менялся относительно v0.35.83.
Миграций нет.
