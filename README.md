Contrast Finance Backend v0.35.85 — changed files only

Фикс переключения месяца/года в кабинете менеджера.

Точная причина:
- select месяца/года менял значение правильно
- но если `/manager-dashboard?month=...` падал или возвращал не тот набор,
  frontend не очищал старый dashboard
- поэтому старые мероприятия оставались на экране, и визуально казалось,
  что месяц не переключился

Исправлено:
1. Frontend теперь перед рендером менеджерского dashboard сам фильтрует события по выбранному месяцу:
   `event.event_date.slice(0, 7) === selectedMonth`

2. Если backend по какой-то причине вернул лишние мероприятия,
   они не попадут в список выбранного месяца.

3. Если `/manager-dashboard` упал,
   frontend больше не оставляет старый экран,
   а показывает пустой dashboard выбранного месяца.

4. При смене месяца/года выбранная карточка очищается сразу,
   ещё до завершения загрузки.

5. В запросы manager-dashboard/payment-requests добавлен cache-buster,
   чтобы браузер/прокси не могли вернуть старый ответ.

Backend не менялся относительно v0.35.84.
Миграций нет.

Проверки:
- app.js прошёл node --check
- Python-файлы компилируются
