Contrast Finance Backend v0.35.63 — changed files only

Первый аккуратный слой кнопки `Оплатить` в кабинете менеджера.

Точная механика:
- кнопка `Оплатить` открывает отдельную компактную модалку
- в модалке выбирается позиция из сметы:
  - обычные позиции
  - координатор
  - ЗП менеджера, если она больше 0
- по выбранной позиции показывается:
  - Факт/смета
  - Оплачено
  - Остаток
- если Факт пустой, база остатка берётся из суммы по смете
- сумма заявки по умолчанию = остаток

Способы оплаты:
1. Нал
   - дополнительных полей нет
   - за позицией фиксируется способ `cash`

2. На карту
   - требуется номер карты, backend валидирует 16 цифр
   - за позицией фиксируется способ `card`

3. По счету
   - БИН/ИИН обязателен
   - перед созданием заявки позиция сохраняется
   - запускается `/event-items/{id}/tax/check`
   - КГД записывает в позицию:
     - БИН
     - налоговый статус
     - НДС
     - вычеты
   - комментарий заявки автоматически заполняется юрназванием из КГД, если оно пришло
   - при повторной заявке по этой же позиции БИН и способ оплаты подставляются автоматически

4. Самозанятый
   - КГД не запускается
   - фамилия обязательна
   - за позицией фиксируется:
     - способ `self_employed`
     - налоговый статус `self_employed`
     - вычеты 10%
     - фамилия в internal_note: `Самозанятый: Фамилия`
   - при повторной заявке фамилия подставляется автоматически

ЗП менеджера:
- создаётся через существующий endpoint:
  `/events/{event_id}/manager-salary/payment-requests`
- доступные способы: Нал / На карту

Backend:

- для self_employed snapshot заявки теперь сохраняет tax_status и deduction_amount
- schema payment_request добавлена в changed-only, чтобы route был согласован
