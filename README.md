Contrast Finance Backend v0.35.9 — changed files only

Исправления калькуляции и форматирования:

Frontend:
- поле `Факт` во внутренней смете отображается с пробелами по тысячам/миллионам
- убраны лишние `.00` в поле `Факт`
- у позиции `Координатор` факт автоматически показывается как 50% от суммы по смете
- факт координатора заблокирован для ручного редактирования
- при сохранении черновика факт координатора записывается как 50% от сметы

Backend calculator:
- внутренние налоги 12% / 5% считаются от клиентской базы без НДС:
  позиции + агентская комиссия
- агентская комиссия участвует в доходной базе менеджера
- координатор по-прежнему не участвует в базе 21%
- компания получает 50% координатора отдельной добавкой после расчёта менеджера
- `tax_base_amount` теперь отдаёт реальную базу налога без НДС

Почему раньше было неверно:
- налоги считались только от обычных позиций без координатора и без агентской комиссии
- доход компании считался от regular_external_total и не включал агентскую комиссию
- из-за этого база менеджера часто уходила в минус, поэтому `Итого менеджеру` становилось 0

index.html подключает app.js/styles.css с ?v=0.35.9

Миграций нет.
