Contrast Finance Backend v0.35.32 — changed files only

Верхние кнопки карточки мероприятия: Удалить / Передать / Соавтор.

Backend:
- DELETE `/events/{event_id}`:
  - удаляет только `draft` / `revision`
  - если есть активные заявки на оплату — не удаляет
  - отменённые / отклонённые заявки не блокируют удаление
  - пока в таблице `events` нет поля `is_deleted`, безопасное удаление сделано как `status = cancelled`
- POST `/events/{event_id}/transfer`:
  - передаёт мероприятие другому менеджеру
  - меняет manager_id и department_id
  - очищает соавторство
- POST `/events/{event_id}/coauthor`:
  - добавляет соавтора
  - создаёт shares 50/50
- POST `/events/{event_id}/coauthor/remove`:
  - удаляет соавторство
  - менеджер, который нажал, забирает мероприятие себе
- GET `/events/action-managers`:
  - список активных менеджеров для модалок Передать/Соавтор
- соавторы получили право видеть и редактировать мероприятие
- manager dashboard:
  - показывает мероприятия, где менеджер основной или соавтор
  - в личный план считает 50% дохода/ЗП менеджера при соавторстве
  - отдаёт coauthor_name / active_payment_requests_count для frontend

Frontend:
- кнопка `Удалить` вызывает удаление
- если по мероприятию есть активные заявки — кнопка `Удалить` disabled
- кнопка `Передать` открывает список менеджеров
- кнопка `Соавтор` открывает список менеджеров
- при соавторстве кнопка меняется на `Удалить соавтора`
- в миникарточке добавляется синий бейдж `Соавтор — Имя менеджера`
- app.js проверен через `node --check`

Важно:
- разделение в общем плане компании остаётся 100% через событие
- в manager dashboard доход делится 50/50
- department/admin split можно отдельно перепроверить на следующем шаге по дашбордам руководителей

index.html подключает app.js/styles.css с ?v=0.35.32

Миграций нет: используется уже существующая таблица event_shares.
