Contrast Finance Backend v0.35.44 — changed files only

Фикс второго нажатия КГД.

Точная причина:
- первый клик по КГД часто приходился на новую tmp-строку
- при сохранении `saveDraftItems()` менял id строки:
  tmp-... → настоящий database id
- но DOM-кнопка и строка в таблице всё ещё жили со старым `data-item-id="tmp-..."`
- результат КГД не всегда попадал обратно в видимую строку
- после любого действия карточка перерисовывалась, DOM получал настоящий id,
  и второй клик уже работал нормально

Исправлено:
- КГД больше не запускается на tmp-id
- перед КГД всегда сначала сохраняется текущая смета:
  `saveDraftEvent()` + `saveDraftItems()`
- если строка была tmp, используется та же ссылка item, у которой после сохранения уже настоящий id
- удалён старый поиск “последней строки” / поиск по BIN
- после КГД карточка перерисовывается, чтобы DOM получил настоящий data-item-id и кнопка стала ✎
- app.js проверен через node --check

Миграций нет.
