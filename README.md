Фикс фиксации способа оплаты за позицией после создания заявки.

Точные причины:
1. Модалка оплаты брала позиции через копии `{...item}`.
   Из-за этого при выборе `Самозанятый` менялась копия, а не настоящая позиция черновика.
   Заявка создавалась, но в смете способ оплаты/вычеты/internal_note не закреплялись.

2. `itemPayloadForSave()` сохранял `tax_check_status` только для `invoice`.
   Для самозанятого теперь явно сохраняется `tax_check_status = self_employed`.

Исправлено:
- позиции в модалке оплаты теперь являются реальными объектами черновика, а не копиями
- при создании заявки `Самозанятый` сразу онлайн записывает в смету:
  - payment_method = self_employed
  - tax_check_status = self_employed
  - deduction_amount = 10%
  - internal_note = `Самозанятый: Фамилия`
- строка сметы, итоги и миникарточка обновляются сразу после создания заявки
- после создания заявки она добавляется в локальный список заявок, чтобы UI сразу заблокировал позицию

Блокировки:
- если по позиции есть активная заявка не в статусе `rejected`, способ оплаты в смете нельзя менять
- если активная заявка по счету, БИН нельзя менять
- карандашик БИН становится disabled
- при отмене заявки и обновлении данных блокировка снимается

Backend не менялся.
Миграций нет.

Проверки:
- app.js прошёл node --check
- Python-файлы компилируются
