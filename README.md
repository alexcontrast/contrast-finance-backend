# Contrast Finance v0.5.44

Точечный патч запрещает менеджерам создавать новые заявки на оплату по мероприятиям, которые уже полностью архивны: мероприятие принято и деньги в кассе.

Защита работает в трёх слоях:

1. Backend отклоняет создание заявки менеджером.
2. Telegram-бот не показывает архивные мероприятия в сценарии новой заявки и повторно проверяет статус перед записью.
3. Веб-кабинет отключает кнопку `Оплатить` у архивного мероприятия.

Администратор по-прежнему может выполнять свои служебные действия. Миграций БД нет.
