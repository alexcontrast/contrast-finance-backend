Contrast Finance Backend v0.35.68 — changed files only

Фикс связки `Самозанятый → фамилия → вычеты` и создания новой позиции из модалки оплаты.

Точные причины:
1. Если старая заявка самозанятого уже была создана, но `internal_note` позиции не успел сохраниться,
   модалка блокировала поле фамилии, но не могла взять фамилию из позиции.
   Теперь фамилия берётся не только из `internal_note`, но и из активной заявки:
   `contractor_name_snapshot` / `comment`.

2. Статус `self_employed` должен всегда автоматически означать:
   - payment_method = self_employed
   - tax_check_status = self_employed
   - vat_amount = 0
   - deduction_amount = 10% от Факта, а если Факт пустой — от суммы сметы
   - internal_note = `Самозанятый: Фамилия`, если фамилия известна

3. Новая позиция из модалки оплаты раньше создавала:
   - Смета = сумма заявки
   - Факт = сумма заявки
   Теперь создаёт:
   - Смета/цена = 0
   - Факт = сумма заявки

Исправлено:
- повторная заявка по уже закреплённому самозанятому подтягивает фамилию из активной заявки
- кнопка `Создать заявку` больше не требует заново вводить фамилию, если она уже есть в активной заявке
- вычеты по самозанятому автоматически восстанавливаются/пересчитываются при отображении, сохранении и создании заявки
- новая позиция из модалки заполняет только `Факт`, а не цену по смете

Backend не менялся.
Миграций нет.

Проверки:
- app.js прошёл node --check
- Python-файлы компилируются
