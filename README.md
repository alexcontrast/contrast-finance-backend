Contrast Finance Backend v0.35.72 — changed files only

Правило защиты удаления мероприятия и понятные tooltip для удаления.

Изменено:
1. Мероприятие нельзя удалить, если по любой его позиции есть хотя бы одна активная заявка на оплату.
2. Активная заявка = любой статус, кроме `rejected`.
3. Кнопка `Удалить` у мероприятия становится неактивной.
4. При наведении на кнопку удаления мероприятия показывается:
   `Есть активные заявки на оплату`
5. При наведении на заблокированную кнопку удаления позиции теперь тоже показывается:
   `Есть активные заявки на оплату`
6. Добавлена дополнительная защита в функции удаления мероприятия:
   даже если попытаться вызвать удаление вручную, удаление не выполнится.

Backend не менялся.
Миграций нет.

- Python-файлы компилируются
