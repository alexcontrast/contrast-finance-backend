Contrast Finance Backend v0.35.74 — changed files only

Причина бага:
- защита удаления была в основном по frontend-state (`state.managerPaymentRequests`)
- на другом устройстве карточка могла открыться без актуального списка заявок
- из-за этого кнопки удаления могли стать активными
- нужна обязательная серверная проверка, чтобы даже при сбое интерфейса удаление не проходило

Исправлено:
1. Frontend:
   - перед отрисовкой карточки мероприятия теперь принудительно подтягиваются заявки:
     `/events/{event_id}/payment-requests`
   - кнопки удаления позиции и мероприятия сразу получают актуальную блокировку
   - активная заявка = любой статус, кроме `cancelled` и `rejected`

2. Backend:
   - добавлен/зафиксирован серверный guard на удаление позиции:
     если по позиции есть активные заявки — DELETE `/event-items/{item_id}` возвращает ошибку
   - добавлен/зафиксирован серверный guard на удаление мероприятия:
     если по мероприятию есть активные заявки — DELETE `/events/{event_id}` возвращает ошибку
   - это сработает даже если frontend когда-нибудь снова ошибочно сделает кнопку активной

3. Tooltip остался:
   `Есть активные заявки на оплату`

Миграций нет.

Проверки:
- app.js прошёл node --check
- Python-файлы компилируются
