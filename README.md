Contrast Finance Backend v0.35.75 — changed files only

Пояснение:
- заявки действительно сразу пишутся в базу
- проблема была не в базе, а в том, что карточка мероприятия не всегда догружала заявки перед отрисовкой кнопок
- с v0.35.74 карточка перед отрисовкой подтягивает `/events/{event_id}/payment-requests`
- backend guard на удаление позиции/мероприятия оставлен: даже если frontend ошибётся, сервер не даст удалить при активных заявках

Добавлено:
1. В карточке менеджера после кнопки `Оплатить` добавлена кнопка:
   `Мои оплаты`

2. `Мои оплаты` открывает модалку по текущему мероприятию:
   - список заявок менеджера по этому мероприятию
   - позиция
   - сумма
   - способ оплаты
   - налоговый статус
   - комментарий
   - статус

3. Менеджер может отменить заявку из этой модалки, если она ещё не оплачена:
   - нельзя отменить `paid`
   - нельзя отменить `cash_received`
   - нельзя отменить уже `rejected`
   - остальные статусы можно отменить

4. После отмены:
   - заявка обновляется на сервере
   - список заявок в модалке обновляется
   - карточка мероприятия перерисовывается
   - блокировки удаления/способов оплаты пересчитываются

Backend route guards из v0.35.74 сохранены.
Миграций нет.

Проверки:
- app.js прошёл node --check
- Python-файлы компилируются
