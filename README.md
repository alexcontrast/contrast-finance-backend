Contrast Finance Backend v0.37.10 — changed files only

Фикс блокировки редактирования менеджером мероприятия со статусом `На проверке`.

Точная причина:
- локальный черновик карточки менеджера мог хранить старый status=draft/revision;
- после отправки на проверку свежий status=review не всегда перезаписывал этот локальный черновик;
- из-за этого карточка снова считалась редактируемой.

Исправлено:
1. getDraftEvent() теперь всегда обновляет из backend/dashboard служебные поля:
   - status
   - money_status
   - coauthor/share/payment counters

2. Если status=review:
   - поля мероприятия становятся disabled;
   - поля сметы становятся disabled;
   - кнопка `Сохранить черновик` становится disabled;
   - кнопка `Отправить Саше` становится disabled;
   - обработчики кликов и input/change дополнительно проверяют актуальный статус.

3. Backend-защита:
   - менеджер не может PATCH-ить мероприятие, если оно не draft/revision.
   - это защищает от обхода frontend.

Миграций нет.

Проверки:
- app.js прошёл node --check
- Python-файлы компилируются
