Contrast Finance Backend v0.33.0 — changed files only

Единый калькулятор мероприятия приведён к финальной схеме.

Главное:
- один серверный калькулятор в app/services/event_calculator.py
- кабинет менеджера и админка должны использовать /events/{id}/summary как один источник правды
- админка показывает snapshot/summary, а не считает формулы сама

Новые явные поля в /events/{id}/summary:
- turnover_with_vat — оборот / клиентская сумма мероприятия
- client_vat_amount — НДС с клиентской сметы
- contractor_vat_credit — НДС подрядчиков в зачёт
- vat_to_pay — НДС к оплате = client_vat_amount - contractor_vat_credit
- tax_rate_percent — ставка налогов: 12 / 5 / 0
- tax_base_amount — база налога
- taxes_total — сумма внутренних налогов

Совместимость:
- external_total сохранён как старое имя оборота
- vat_total теперь равен vat_to_pay, чтобы старый frontend не показывал НДС подрядчиков вместо НДС к оплате

Формулы:
- ip_contrast_event:
  - оборот считается суммой внешней сметы с НДС
  - клиентский НДС = оборот * 12 / 112
  - налог = 12% от регулярной сметы без НДС
- our_no_vat:
  - клиентский НДС = 0
  - налог = 12% от регулярной сметы
- simplified:
  - клиентский НДС = 0
  - налог = 5% от регулярной сметы
  - банк+налоги сверху учитываются отдельной суммой simplified_bank_tax_amount
- cash:
  - налог = 0
  - НДС = 0

Подрядчики:
- По счету: НДС берётся в зачёт, вычеты берутся из КГД/позиции
- Самозанятый: НДС 0, вычеты 10%
- Налик/На карту: НДС 0, вычеты 0

Координатор:
- исключён из базы 21% менеджера
- 50% координатора добавляется в финальный доход компании после ЗП менеджера

ЗП менеджера:
- считается сервером от manager_salary_base
- item_type=manager_salary исключён из расчёта прибыли, чтобы ЗП не вычиталась дважды

Frontend:
- модалка мероприятия берёт Оборот из turnover_with_vat
- карточка НДС берёт vat_to_pay
- карточка Налоги берёт taxes_total и tax_rate_percent
- index.html подключает app.js/styles.css с ?v=0.33.0

Миграций нет.
