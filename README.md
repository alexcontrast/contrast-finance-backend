Contrast Finance Backend v0.35.83 — changed files only

Фикс проверки БИН в модалке оплаты по счету.

Точная причина:
- frontend после проверки БИН ждал поле `taxResult.tax_status`
- backend проверки БИН возвращает основной статус в поле `tax_check_status`
- из-за этого проверка проходила, но модалка считала, что статус не пришёл
- БИН не фиксировался в локальной позиции, и кнопка `Создать заявку` снова требовала проверку

Исправлено:
- модалка теперь принимает статус из:
  - `tax_check_status`
  - или fallback `tax_status`
- после успешной проверки жёстко фиксируется:
  - payment_method = invoice
  - iin_bin
  - iin_bin_locked = true
  - tax_check_status
  - vat_amount
  - deduction_amount
- после проверки позиция дополнительно сохраняется в смету
- поле БИН, способ оплаты и кнопка `Проверить` сразу становятся неактивными
- кнопка `Создать заявку` больше не должна требовать повторную проверку после успешной проверки

Backend не менялся относительно v0.35.82.
Миграций нет.

Проверки:
- app.js прошёл node --check
- Python-файлы компилируются
