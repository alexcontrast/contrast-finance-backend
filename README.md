Contrast Finance Backend v0.35.43 — changed files only

Фикс первого клика по кнопке КГД.

Точная причина:
- на кнопку КГД были навешаны два обработчика одновременно:
  1) старый прямой `addEventListener("click", ... checkTaxForItem(...))`
  2) новый контролируемый `bindTaxButtons()` с loader `...`
- при первом клике запускались две параллельные проверки КГД
- на новой tmp-строке это давало гонку:
  - один обработчик сохранял tmp-строку и менял id
  - второй продолжал работать со старым tmp-id
  - визуально кнопка оставалась ✓, вычеты не обновлялись
- после любого изменения карточка перерисовывалась, строка получала настоящий id, и второй клик уже проходил нормально

Исправлено:
- удалён старый прямой обработчик `[data-check-tax-item]`
- оставлен только один обработчик через `bindTaxButtons()`
- добавлена защита от двойного клика `data-kgd-loading`
- index.html cache-bust до ?v=0.35.43

Проверено:
- в app.js больше нет прямого вызова `checkTaxForItem(button.getAttribute(...))`
- `bindTaxButtons()` вызывает КГД один раз
- app.js прошёл node --check
- Python-файлы компилируются

Миграций нет.
